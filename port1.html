UI UX Case Study : Car to Go

A mobile application that hepls user to rent a car.


About this Project

This project is part of the Independent Study at Binar Academy through the Kampus Merdeka program organized by Kementrian Pendidikan, Budaya, Riset, dan Teknologi. This project takes about 1 month in Sept 2022 for its process.

My Role

My role on this project is as a UI UX Designer and Researcher.

Background Research

The covid-19 pandemic has ended, many people who feel confined at home finally feel free. Various ways are done to get revenge with this, one of which is traveling with family. But there is a major problem that can occur, which is the limitation of car rental to reach the vacation destination.

The limitations of car rental owners, which are only known as word of mouth, make them cancel their intentions when there are no cars available.

Goals
Target User
Helping users who want to travel more easily find a car rental place according to their wishes. Male or Female, 21-30 years, who likes to travel.

Design Process

Research Insight

Empathize with user using the User Interview method to understand directly the problems that user feels. Before the interview is conducted, several questions are first made that are asked to the user. There are several problems experienced by users, including:

1. Users often find it difficult to find a rent car vendor at a sudden time.

2. Lack of information regarding the place of rental car and the criteria for the desired car.

3. Spend a lot of time looking for information regarding guaranteed car rental places.

I have interviewed 3 users with different backgrounds, so that conclusions can be drawn about which users can use my application. After brainstorming and creating emphaty maps and affinity diagrams, a user persona is formed.

High Fidelity and Prototype

In this interface design, I use a combination of yellow, black, and white. Not for the reason of using the color, I chose that color in the end because the combination of colors has the meaning that it can increase cheerfulness and confidence when the user uses the application.

I try to make it as simple as possible and according to the persona that has been chosen so that users can comfortably run the Car To Go application. There are several High Fidelity design pages that can you see it, and you can also see the prototyping of the Car To Go application here

Usability Testing

Usability Testing is a test that aims to find the best and most effective way for users to complete a given task after they run a real prototype based on user feedback.

Here, I use the Moderated Usability Testing method with 3 participants. Using the Zoom, Maze, and Userpoll applications as a means of implementing UT. Participants were directed to work on several scenarios that had been prepared.

- First Scenario: Exploration. As a user, please explore the Onboarding page and login to the Car to Go application.

- Second scenario: Find Car. As a user, you are asked to find a car rental without a driver in Surabaya from September 9 to September 11, 2022.

- Third scenario: Choose Car. As a user, please select a jeep car from the available vendors and continue the order process until payment is complete.

System Usability Scale

This stage is a test to measure how feasible the application design that has been designed to be implemented. There are 10 questions that need to be answered by the user. I use the Userpoll platform to get SUS results quickly.

Based on the result of the System Usability Scale, Car to Go got 74,2.

Conclusions

Based on the survey results from participants through the results of the System Usability Scale, this application design can be implemented because the numbers show above the minimum limit. However, it is still necessary to add some improvements from the feedback that has been given by participants.
