UI UX Case Study:

Redesign M.Tix App


About this Project

This project is part of the Challenge on Independent Study of Kampus Merdeka at Binar Academy. Takes about 1 week on October 14-20, 2022 for the process of this project.

Background Research

M.Tix, which is the official application from Cinema XXI which displays online booking ticketing, received a low rating on the Appstore. With this rate and the large number of M.Tix application users, it became my own interest to analyze what needs to be improved from the application.

Research Method

I use the Desk Research method to work on this redesign project because the collection time limit is quite short so that using this method can save time and costs. The data collection used is primary and secondary data.

The primary data here is taken from the results of interviews with users and collects insights that are used to validate secondary data. The secondary data in this project uses previously published data, namely in the M.Tix application feedback on the Appstore or Playstore.

Interview

M.Tix, which is the official application from Cinema XXI which displays online booking ticketing, received a low rating on the Appstore. With this rate and the large number of M.Tix application users, it became my own interest to analyze what needs to be improved from the application.

Insight

Based on data from interviews that have been conducted, I get several insights which include:

1. The appearance that has always remained unchanged and boring when running applications.

2. Payment methods can only be through Sakuku and M.Tix

3. There is no choice of dates for booking movies.

4. The scan icon on the M.Food page is annoying.

User Persona

Pain Point and Room for improvements

Homepage

- Pain Point: Reduces whitespace and makes 1 of the existing 2 buttons play and upcoming.

- Room for improvements: Reduces whitespace and combines 2 playing and upcoming buttons

Movie Details

- Pain Point: The 2 buttons (Playing at and buy ticket) have the same flow until the end, namely booking. There is no date option.

- Room for improvements: Brings together the Playing at and Buy Ticket buttons that have the same flow, adding a choice of dates.

M.Food

- Pain Point: T: Search field that is rarely to never used, inappropriate placement of qr scan icon.

- Room for improvements: Replaces the search field with a place to scan the QR.
